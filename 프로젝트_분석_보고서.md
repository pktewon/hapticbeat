## 1. 프로젝트 개요

### 🎯 HapticBeat이란?
**음악을 시각과 청각을 넘어 촉각으로 경험하게 하는 혁신적인 모바일 애플리케이션**

### 핵심 컨셉
- 음악 파일의 **비트, 리듬, 강도**를 실시간으로 분석
- 분석 결과를 **햅틱(진동) 피드백**으로 변환
- 음악 재생과 **완벽하게 동기화**된 촉각 경험 제공
- 청각 정보를 **촉각 정보로 변환**하는 감각 확장 기술

### 프로젝트 상태
- **완전 기능 구현** 완료
- **프론트엔드(React Native)** + **백엔드(FastAPI)** 풀스택 구조
- **Docker 기반** 배포 환경 구축
- **실시간 음악 분석** 알고리즘 탑재

## 2. 핵심 기능 및 기술

### 🎼 주요 기능

#### 1️⃣ 지능형 음악 분석
- **다양한 포맷 지원**: MP3, WAV, M4A, FLAC, OGG
- **고급 오디오 분석 기술**:
  - 🎵 **비트 검출 (Beat Detection)**: 음악의 박자 자동 인식
  - 📊 **RMS 에너지 분석**: 음악의 강도/진폭 측정
  - 🎯 **Onset Strength**: 타격감 및 음의 시작점 감지
  - 🌈 **스펙트럴 센트로이드**: 음색의 밝기 분석

#### 2️⃣ 정밀한 햅틱 생성
- **시간, 강도, 지속시간**을 포함한 세밀한 햅틱 데이터
- **3단계 진동 강도**:
  - 💪 Heavy (0.8-1.0): 강한 비트
  - 👍 Medium (0.5-0.8): 중간 비트
  - 👌 Light (0.2-0.5): 약한 비트
- **동적 지속시간 조절**: 음악 강도에 따라 진동 길이 자동 조절

#### 3️⃣ 완벽한 동기화
- **실시간 재생 추적**: 50ms 단위 정밀 동기화
- **시크(Seek) 기능**: 원하는 구간으로 이동 시 햅틱 자동 재정렬
- **백그라운드 재생** 지원

#### 4️⃣ 직관적인 UI/UX
- 🎨 **현대적인 다크 테마**
- 📱 **모바일 최적화** 인터페이스
- 🎚️ **시각적 진행 표시**: 햅틱 이벤트 카운터, 시간 표시
- 🎛️ **직관적인 컨트롤**: 재생/일시정지, 정지, 시크바, 햅틱 테스트

## 4. 필요성 및 가치 제안

### 🌟 왜 HapticBeat이 필요한가?

#### 문제 인식

##### 1️⃣ 청각 장애인의 문화적 소외
```
💔 현재 상황:
- 음악 콘서트, 클럽, 음악 축제 등에서 소외
- 음악 감상이라는 보편적 문화 경험 불가능
- 음악 교육 및 리듬 학습의 어려움
- 제한된 엔터테인먼트 옵션
```

##### 2️⃣ 단일 감각에 의존하는 기존 음악 경험
```
🎧 전통적 음악 감상:
- 청각에만 의존
- 제한적인 몰입도
- 단조로운 감상 방식
```

##### 3️⃣ 접근성 기술의 부재
```
🚫 기존 솔루션의 한계:
- 시각화(스펙트럼 분석): 청각 장애인에게 의미 제한적
- 자막: 가사만 전달, 리듬과 감정 전달 불가
- 수화 통역: 음악 자체의 경험 불가
```

### 💡 HapticBeat의 솔루션

#### 1. **포용적 접근성 (Inclusive Accessibility)**
> "모든 사람이 음악을 경험할 권리가 있습니다"

- ✅ 청각 장애인도 음악의 리듬, 비트, 감정을 **촉각으로 체험**
- ✅ 문화적 장벽 해소 및 **사회적 포용성** 증대
- ✅ UN 장애인권리협약의 **문화 접근권** 실현

#### 2. **다감각 경험 (Multisensory Experience)**
> "음악은 듣는 것이 아니라 느끼는 것입니다"

- 🎯 청각 + 촉각 결합으로 **입체적 음악 경험**
- 🎯 뇌의 여러 영역 동시 자극으로 **몰입도 극대화**
- 🎯 과학적으로 입증된 **다감각 통합의 효과**

#### 3. **기술적 혁신 (Technical Innovation)**
> "AI 기반 음악 분석과 햅틱 기술의 융합"

- 🔬 **librosa** 라이브러리 기반 고급 오디오 분석
- 🔬 **실시간 처리** 및 정밀 동기화
- 🔬 **머신러닝 알고리즘** 적용 가능한 확장 가능 구조

#### 4. **보편적 가치 (Universal Value)**
> "장애인 보조 기술이 아닌, 모두를 위한 경험"

- 🌍 청각 장애인: 필수 도구
- 🌍 일반 사용자: 새로운 차원의 음악 감상
- 🌍 전문가: 창작 및 교육 도구

### 🎯 핵심 알고리즘

#### 햅틱 생성 알고리즘
```
입력: 음악 파일
출력: 햅틱 이벤트 배열

1. 오디오 로드 (librosa)
   └─> 샘플링 레이트, 시간 영역 데이터 추출

2. 다중 특징 추출
   ├─> 비트 검출: 음악의 주요 박자
   ├─> RMS 에너지: 음의 강도
   ├─> Onset Strength: 음의 시작점 강도
   └─> 스펙트럴 센트로이드: 음색의 밝기

3. 특징 융합 (Feature Fusion)
   강도 = (RMS × 0.6) + (Onset × 0.3) + (Spectral × 0.1)
   └─> 가중 평균으로 최종 햅틱 강도 계산

4. 필터링 및 정규화
   ├─> 약한 이벤트 제거 (임계값 0.2)
   ├─> 0.0 ~ 1.0 범위로 정규화
   └─> 중복 이벤트 제거 (0.1초 이내)

5. 지속시간 계산
   duration = 0.05 + (intensity × 0.1)
   └─> 강도에 비례하는 진동 길이

6. 햅틱 이벤트 생성
   └─> { time, intensity, duration } 배열
```
